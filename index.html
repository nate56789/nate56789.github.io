Hello and welcome to this interactive chapter on Computational Social Choice. What is Computational Social Choice? Well ...
Definitions

Computational Social Choice is the field combining many different areas of study. These areas include theoretical computer science, social choice theory, and more.

Theoretical Computer Science (in very brief summary) is a branch of computer science that focuses more on the mathematical components of computer science rather than just applications of these components.

Social Choice Theory is a type of methodology for making equitable decision-making. Specifically, these decisions are based on user preferences, opinions, and interests and potentially other characteristics. This decision that is reached needs to be more or less collective.

Hopefully those definitions gave you some idea of where we are going in this chapter. Computational social choice is seen all over the place and this chapters will show you some of those examples as well as the underlying mathematical notion of how those examples can be expressed. Mainly, this chapter will be concerned with making decisions in elections of various forms and analyzing the pros and cons of different election rules. Let us begin with the simplest definition which is that of a voter.
Definitions
A Voter is simply an entity who votes in an election.
This definition directly follows from the real-world concept of a voter. The next few definitions will similarly follow from real-world concepts.
Definitions
A Candidate/Alternative is an entity that can be voted for in an election.

A Preference Ballot/Profile/List is a ranking of the candidates.

Each voter constructs a preference list.
Now that we established how an election can be described, we can discuss different rules in these elections and the pros and cons of each type of rule.
Definitions (Voting Rules)
The Plurality Rule for voting is when we only consider the top-ranked candidate for each voter. Then the candidate that is top-ranked the most times is the winner of the election.
Real Life Example
The plurality rule is essentially what the United States uses for its elections. Each state lists its first preference choice and then across all states the candidate with the most first choice across each state (multiplied by some electoral college weight for population) is the victor. Note that this does not necessarily imply that the winning candidate got the majority of the votes/is the most preferred.
Example
Consider the following table of voting where the first row indicates the number of people with that permutation preference list for those candidates.
2 4 4 4
Alice Alice Bob Charlie
Bob Charlie Charlie Alice
Charlie Bob Alice Bob
Then Alice gets 6 first choice votes, Bob gets 4, and Charlie gets 4. So, Alice would be the winner. However, note that Alice would not win a head to head with both Bob and Charlie even though she wins against both at the same time. Consider Alice vs Charlie. Then Alice would get $(2+4) = 6$ first place votes but Charlie would get $(4+4) = 8$ first place votes. So, Charlie would have won if Bob was not present. We will explore what this property is called below.
Definitions (Voting Rules)
The Instant Runoff/Plurality With Elimination for voting is when each round, we eliminate the candidate with the least first place votes. This continues until a majority is reached for a specific candidate. That candidate is declared the winner of the election.
Real Life Example
This voting rule is actually used for the Australian House of Representatives election.
Example
Using a similar table to previous example
1 2 4 6
Alice Alice Bob Charlie
Bob Charlie Charlie Alice
Charlie Bob Alice Bob
Note that there are a total of 13 voters so it takes 7 votes to reach a majority. In the first round Charlie gets 6 first place votes so a majority is not reached. Therefore we eliminate the person with least first-place votes which in this case would be Alice. Then in the next round with just Charlie and Bob, Charlie gets $2+4+6 = 12$ first-place votes and Bob would get just 1 first-place vote. Therefore Charlie would be declared the winner since he got the majority.
Definitions (Voting Rules)
The similar Plurality With Runoff Rule for voting is when you take the first and second place winner based on the number of first-place votes and pit them against each other.
Example
For the previous table Alice has $3$ first place votes, Bob has $4$ and Charlie has $6$. So since Bob and Charlie are in second and first place respectively, they would go against each other and the end result would be the same as in the previous example.
Definitions (Voting Rules)
The Borda Count voting rule is another voting rule where for each voter's preference list, 1 point is given to the last place candidate, 2 to second to last, 3 to third to last, and so on. Then the candidate with the most amount of points at the end is declared the winner.
Example
Consider the following table of voting where the first row indicates the number of people with that permutation preference list for those candidates.
2 4 4 5
Alice Alice Bob Charlie
Bob Charlie Charlie Alice
Charlie Bob Alice Bob
In this table there are 3 candidates, so first-place gets 3 per voter, second-place 2, and third-place 1. Then for this table Alice gets $2*3 + 4*3 + 1*4 + 2*5 = 6+12+4+8=32$ for column 1,2,3,4 respectively. Using the same method Bob gets $2*2 + 4*1 + 4*3 + 5*1 = 25$ and finally Charlie gets $2*1 + 4*2 + 4*2 + 5*3 = 33$. So Charlie would be declared the winner. Note that if we were using the plurality rule then Alice would be declared the winner.
Real Life Example
The Borda Count is used all over the place. One instance is the selection of the MVP in Major League Baseball.
Definitions (Voting Rules)
The last voting rule we will discuss is Copeland's Method. In this method, every possible one on one match-up is considered. In each match-up, the candidate that is preferred by more voters to the other one gets a point. Then the candidate with the most points across all the pairwise match-ups (election with two people) is declared the winner.
We talked about different rules, but now we will discuss the different voting properties that exist, which we may want to satisfy, and which of the above rules satisfy which property.
Definitions (Voting Rule Property)
We say a voting rule is Condorcet Consistent if a candidate, when put against any of the other candidate in a one-on-one, is preferred by more voters, then that candidate is the winner. Note that the converse need not necessarily be true.
Exercise
Prove Copeland's Method is Condorcet Consistent

This method is by definition Condorcet Consistent. Assume that there is a candidate, say $X$ that in a one-on-one is preferred to every candidate. Then in every pairwise election involving $2$, we see $X$ will win and receive a point. Therefore, $X$ by construction will receive the most points and therefore will be declared the winner. Therefore Copeland's Method is Condorcet Consistent.
Definitions (Voting Rule Property)
We say a voting rule has the Monotonicity Criterion if voters shift their preference lists to prefer a candidate more, say $X$, that should not decrease the chance for $X$ to win. Intuitively this is something we would definitely want in an election.
Definitions (Voting Rule Property)
We say a voting rule has the Majority Criterion if the candidate declared the winner of the election necessarily has the largest number of first place votes.
Definitions (Voting Rule Property)
The Independence of Irrelevant Alternatives Criterion essentially states if a non-winning candidate is removed from the ballot (and the preference list for the voters stays the same modulo the removed candidate), then the winner of the election should not change.
There are also other ways to vote besides a preference list for each voter. One example is approval voting.
Definitions (Voting Style)
Approval Voting is different from preference list voting in that instead of each voter ranking each candidate from most liked to least liked, they instead simply mark which candidates they approve of. In this way each approved candidate for a single voter essentially has the same preference. Then the candidate with the most approvals is declared the winner.
Example
Consider the following table representing an approval vote.
Alice Bob Charlie Dave Edward
Pulp Fiction X X X
Django X X
Once Upon a Time in Hollywood X X X X X

Then the movie Once Upon a Time in Hollywood would win the election as it was approved by 5 voters. Note that this style of voting may be problematic because a voter may really want one candidate to win so then they only approve that single candidate and no one else. This can change the result of the election.

Also note that this style of voting is not necessarily majority consistent as everyone in a voters preference list gets weighted equally so it is definitely not the case that the most preferred candidate is necessarily the winner of the election.

Now we will do some exercises showing the pros and cons of the above voting rules using the different characteristics that we described above.
Exercise
Prove that the plurality rule is not Condorcet Consistent.

Consider the following table of voting where the first row indicates the number of people with that permutation preference list for those candidates.
2 4 4 4
Alice Alice Bob Charlie
Bob Charlie Charlie Alice
Charlie Bob Alice Bob
So, Alice would be the winner here by plurality rules (easy to verify). Consider Alice vs Charlie without Bob. Alice would get $(2+4) = 6$ first place votes but Charlie would get $(4+4) = 8$ first place votes. So, Charlie would have won if Bob was not present rather than Alice. Additionally, Charlie would also have won against Bob by similar logic. So this example shows plurality is not condorcet consistent.
Exercise
Prove that Borda Count is not Condorcet Consistent.

Consider the following table of voting where the first row indicates the number of people with that permutation preference list for those candidates.
1 1 1 1 1
Alice Alice Bob Charlie Derek
Charlie Derek Alice Bob Bob
Bob Bob Charlie Alice Alice
Derek Charlie Derek Derek Charlie
Under Borda count, Alice gets $4+4+3+2+2=15$, Charlie gets $3+1+2+4+1=11$, Bob gets $2+2+4+3+3=14$ and Derek gets $1+3+1+1+4 = 10$. So, Alice would be declared the winner under Borda Count rules. However, note that Bob, in a one-on-one with any of the other candidates, would be declared the winner. So this counterexample shows Borda Count is not Condorcet Consistent.
The above two results are actually very interesting (and surprising) because intuitively one of the two rules is one that you would most likely pick to use if you were doing an election. However, in both cases it may very well occur that the winner you select is not actually preferred by most voters. This can easily be a reason why it frequently seems like a majority of individuals may be upset with election results (when the election is strictly more than 2 candidates). It also exposes what could be considered a flaw in our voting system.
Exercise
Prove that a voting rule having majority consistency does not imply it also has Condorcet consistency

Plurality Voting is majority consistent by definition of picking the candidate with the largest number of first-place votes. However, it was just shown that plurality voting is not necessarily Condorcet Consistent.
Exercise
Prove that a voting rule having Condorcet consistency does imply the majority criterion

This follows from what we want to show. Assume that the voting rule satisfies condorcet consistency. Then we want to show that a candidate having a majority of first place votes implies that the candidate is the winner. If a candidate has the majority of first place votes, then in any pairwise election, that candidate will still have more first-place votes than the other candidate since they have a majority. Then by definition of condorcet consistency, this implies that this candidate would be the winner. Therefore the voting rule would also be majority consistent.
These results are actually very interesting because they show that condorcet consistency is actually stronger than the majority criterion which intuitively may not make sense as they both seem equally strong. The last thing we will talk about is a very interesting theorem about the limitations of preference voting.
Definitions (Preference List Voting Theorem)
Arrow's Impossibility Theorem describes how it is Impossible to construct a voting rule with preference lists that satisfies all three of these properties:
  • If every voter prefers one candidate over another, then that preferred candidate is the winenr
  • The isolation property which really means that if you consider two candidates in isolation, and don't change the preference between them, then the outcome of the election would not change the preference between these two candidates even if other preferences between other candidates changes
  • Non-dictatorship which really means that no single voter can always influence the election outcome. In other words, the votes of many voters are taken into account.
This is surprising because it shows an inherent flaw in this style of voting and how it is provably impossible to get all three of these desired characteristics. This is extremely similar to the database CAP theorem which also highlights how it is impossible to get 3 desired characteristics about databases.
